Tiny calendar icons sprite

28 August 2009

Recently I've been on a mission to minimize the number of HTTP requests made while loading pages on this site. Until yesterday, the archives page was making an HTTP request for each of the tiny calendar icons used on the page. Therefore, up to 31 HTTP requests were required just to retrieve the calendar icons. Not good.

The same result can be achieved with a single HTTP request through the use of a sprite:

Tiny calendar icons sprite, which you're welcome to save and use

One uses a sprite by applying a background image to an element using CSS. One could start with an empty span element:

<span class="day-1"></span>

The following CSS sets the width and height of the element and sets the sprite as its background image:

span.day-1
{
    display: block;
    width: 16px;
    height: 16px;
    background-image: url(/path/to/images/calendar.png);
    background-repeat: no-repeat;
}

By default the top left corner of an element's background image is aligned with the element's top left corner, so at this stage all that's displayed is a 16px by 16px white box:

Calendar sprite applied as background image using default positioning

To have the "1" icon displayed, the element's background position is specified:

span.day-1
{
    display: block;
    width: 16px;
    height: 16px;
    background-image: url(/path/to/images/calendar.png);
    background-repeat: no-repeat;
    background-position: -60px -20px;
}

This drags the background image 60px to the left, and lifts it 20px:

Calendar sprite applied as background image using correct positioning

Here's the final CSS:

span.day
{
    display: block;
    float: left;
    margin: 0.167em 0.5em 0 0;
    width: 16px;
    height: 16px;
    background-image: url(/path/to/images/calendar.png);
    background-repeat: no-repeat;
}

Note that I introduced the class name day to streamline the CSS. This must appear in the HTML, as in this example:

<span class="day day-31"></span>
